# 事先七葛亮

项目还没开始，七葛亮先吵一遍。

**事先七葛亮** 是一人公司的启动前虚拟评审委员会。用户输入一个创业想法、产品方案、合作计划、运营活动或个人重大决策，它会召集七个评审官进行多轮预审，输出风险清单、法律与证据专项、推荐执行方案和启动条件 Checklist。

它特别适合 **一人公司（OPC / One Person Company）**：当一个人同时承担产品、用户验证、获客、交付、客服、财务、法务和复盘时，用七个岗位化评审官补齐团队视角。

## 解决什么问题

很多项目不是死在执行，而是启动前没有被认真反驳过：

- 需求未经验证，却开始开发。
- 获客路径只停留在想象。
- 一个人承担太多角色，范围失控。
- 合同、隐私、版权、证据链被忽略。
- 团队会议只剩“大家觉得没问题”。

这个 Skill 把风险提前暴露出来，让用户在投入真实时间和钱之前，先完成一次高密度预审。

## 七个评审官

| 角色 | 职责 |
| --- | --- |
| 反方审计官 | 拆前提、找逻辑漏洞、识别自嗨叙事 |
| 用户验证官 | 检查目标用户、付费意愿、获客路径和验证证据 |
| 风险预警官 | 预测现金流、时间、平台、交付、团队等高概率翻车点 |
| 范围裁剪官 | 砍功能、压范围、逼出最小可交付版本和低成本实验 |
| 执行总控官 | 把结论变成行动项、截止时间、交付物和验收标准 |
| 约束破局官 | 在资源很少、质疑成立时寻找艰难但可验证的活路 |
| 合规证据官 | 检查合同、隐私、版权、付款、留痕、责任边界 |

## 为什么适合 OPC 赛题

一人公司最大的难点不是“没人”，而是一个人要临时扮演一整支团队。事先七葛亮把缺失的团队视角拆成七个稳定评审官：

- 反方审计：先拆掉自嗨和未经验证的前提。
- 用户验证：只认真实用户、真实场景、真实付费证据。
- 风险预警：计算最坏情况、现金流底线和止损点。
- 范围裁剪：把想做的东西砍成一个最小闭环。
- 执行总控：把模糊建议变成可验收动作。
- 约束破局：在资源很少时找可验证的小活路。
- 合规证据：检查隐私、合同、版权、付款和证据链。

```text
事先七葛亮：我想一个人做一个 AI 简历优化 SaaS。用户上传简历和岗位 JD，系统自动给出简历修改建议、匹配度评分和面试准备清单。先用小红书获客，3 周上线 MVP，预算 5000 元，目标第一个月获得 50 个付费用户。
```

## 输出内容

默认输出五部分：

1. **三轮圆桌讨论**：各自亮刀、互相拆台、收网定论。
2. **项目启动前风险清单**：高/中/低风险、概率、影响、规避动作、验收标准。
3. **法律与证据专项**：责任主体、可能后果、需要补的证据或条款。
4. **推荐执行方案**：P0/P1/P2 行动项、负责人、截止时间、交付物、验收标准。
5. **启动条件 Checklist**：判断现在能不能开干。

完整演示见 [references/opc-demo.md](references/opc-demo.md)。

## 设计原则

- **可执行**：所有建议落到行动项、交付物和验收标准。
- **可验证**：没有证据的假设必须标为“待验证”。
- **不和稀泥**：角色之间可以冲突，冲突本身就是价值。
- **不冒充专业意见**：法律、医疗、财务等高风险内容只做风险提示和材料清单。
- **适合单人启动**：优先考虑砍范围、低成本验证、自动化替代和现金流止损。

## 适用场景

- 一人公司/独立开发者项目启动前评审
- AI 工具、SaaS、课程、咨询、内容、电商项目预审
- 新产品或新功能上线前风险扫描
- 签约合作、投资、转型、换工作等重大决策
- 团队方案评审，替代“大家觉得没问题”的橡皮图章会议
