# 事先七葛亮 · 项目启动前圆桌顾问

> 「项目还没开始，七葛亮已经吵完了——连活路都给你找好了。」

---

## 是什么

**事先七葛亮**是一个 AI Agent Skill，专为项目方案评审而设计。

你把一个项目想法或方案扔进来，它会自动唤醒 7 个性格完全不同的顾问角色，围绕你的方案进行 **至少三轮激烈的圆桌讨论**，把该踩的坑提前踩完，把该吵的架提前吵完。最后输出两份落地文档：**《项目启动前风险清单》** 和 **《推荐执行方案》**。

事后诸葛亮人人会做——这个 Skill 让诸葛亮**在项目开始之前**就亮了。

---

## 为什么要用

大多数项目失败，不是死在执行上，而是死在启动前没有被认真质疑过。

- 老板拍板，没人敢说不
- 团队内部都觉得"应该没问题"
- 没有人从法律、人性、底层逻辑层面系统性地找过坑

七葛亮解决的就是这个问题：**在你投入真实资源之前，先在脑子里打一场仗。**

---

## 七个角色

六个刹车 + 一个方向盘，各司其职、互不重叠：

| 角色 | 唯一职责 | 人格原型 |
|------|----------|---------|
| 🎯 **事先诸葛亮** | 死命挑刺，挖根本原因，绝不给建议 | 王德峰式哲学批判 |
| 🚨 **避坑预警亮** | 预测最可能踩的坑，给出规避方案 | 冯唐务实主义 |
| 📦 **经验打包亮** | 把所有结论打包成可复用模板和 SOP | 李笑来方法论 |
| 🔨 **行动落地亮** | 把结论翻译成有责任人、有截止时间的行动项 | 顶级项目经理 |
| 🔥 **逆风翻盘亮** | 当六亮都说"别做"时，找那条艰难的活路 | 罗永浩逆境思维 |
| 📚 **知识深挖亮** | 从原理层面解释为什么会这样，不给决策建议 | 李永乐科普方法论 |
| ⚖️ **法外狂徒亮** | 用法律和人性逻辑找出普通人看不到的坑 | 罗翔刑法思维 |

> **重要规则**：每个角色严格在自己的职责范围内发言，绝不越界。诸葛亮只挑刺不给方案，翻盘亮只找活路不唱衰，七人的矛盾必须保留——这是讨论的灵魂。

---

## 讨论结构

```
第一轮：各自亮刀（初始判断）
每个角色从自己的视角快速亮出核心观点

第二轮：互相拆台（质疑与碰撞）
角色之间互相质疑、补充、抬杠
翻盘亮 vs 法外狂徒亮 是每轮的情绪高点

第三轮：收网定论（最终立场）
每人给出最终判断，一锤定音
```

三轮讨论结束后，输出：
- 📋 **《项目启动前风险清单》**：高/中/低风险分级，含法律风险专项
- 📋 **《推荐执行方案》**：核心结论 + 优先行动项 + 可复用模板清单 + 启动条件 Checklist

## 怎么使用

### 安装（WorkBuddy Skill）

将本目录放入 WorkBuddy 用户级技能目录：

```
~/.workbuddy/skills/shiqian-qiliang/
```

### 触发

在对话中直接说：

```
事先七葛亮：[你的项目方案]
```

或使用以下任意触发词：

`事先七葛亮` · `帮我把关` · `方案评审` · `项目评审` · `开个圆桌` · `项目挑刺` · `这个方案靠谱吗` · `启动前评审`

### 输入格式

方案粗糙还是详细都可以：

- **一句话**："我想在牡丹园开一家洗车店" → 七葛亮会先帮你把方案补全再开吵
- **一段描述**：几百字的创业想法、产品构想、运营方案
- **完整文档**：七葛亮会直接进入深度分析模式，逐条审查

## 输出示例

```markdown
# 项目启动前风险清单

## 🔴 高风险（必须解决后才能启动）
| # | 风险描述 | 概率 | 影响 | 来源 | 规避方案 |
|---|---------|------|------|------|---------|
| 1 | 选址未确认经营资质（排水许可、消防） | 高 | 致命 | ⚖️法外狂徒亮 | 签租约前先跑审批流程 |
| 2 | 未测算保本单量，现金流断裂风险 | 高 | 致命 | 🚨避坑亮 | 先做保本测算再签租约 |

## 🔥 如果非做不可（逆风翻盘亮）
走豪车精洗小而美路线，月服务 20 台均价 800+，靠技师口碑裂变，
避开快洗的完全竞争市场。
```

## 设计原则

1. **角色辨识度优先**：遮住角色名，你能从说话方式认出是谁——这是质量标准
2. **禁止和稀泥**：七个角色的矛盾必须保留，不能以"大家说得都有道理"收场
3. **翻盘亮的活路必须是艰难的**：世上没有轻松的翻盘，找到的路一定有代价、有条件
4. **三轮讨论不可缩水**：不能用"经过激烈讨论"一句话跳过，每一轮必须完整呈现

---

## 适用场景

- 创业项目启动前的自我质疑
- 新产品/新功能上线前的风险扫描
- 重大决策（投资、签约、转型）前的多角度论证
- 团队方案评审，替代"大家觉得没问题"的橡皮图章会议
- 个人重大决定（换工作、搬城市、开店）的系统性思考
